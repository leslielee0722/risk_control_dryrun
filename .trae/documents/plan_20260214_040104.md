# IP配置文件统一管理计划

## 一、现状分析

### 1. 两个 `.env` 文件配置

| 文件路径 | 当前配置 | 状态 |
| ---------------- | ---------------- | ---- |
| `/.env` | `SERVER_IP=127.0.0.1` | ✅ 本地开发环境 |
| `/development_platform/.env` | `SERVER_IP=127.0.0.1` | ✅ 本地开发环境 |

两个文件配置一致，都是本地开发环境。

### 2. IP配置依赖关系

| 文件 | 读取方式 | 依赖的 `.env` 文件 |
| ---------------- | ---------------- | ------------------- |
| `start.py` | `os.getenv('SERVER_IP')` | 项目根目录 `/.env` |
| `development_platform/run.py` | `load_dotenv()` | `development_platform/.env` |
| `development_platform/app/__init__.py` | `os.getenv('SERVER_IP')` | 由 `run.py` 传递 |
| `cot_synthesis_service.py` | 硬编码 `http://100.100.135.172:8081` | 无依赖（LLM API，保持不变） |

### 3. 当前问题

- 存在两个 `.env` 文件，容易造成配置不一致
- `start.py` 和 `run.py` 各自读取不同的 `.env` 文件
- 需要统一配置源，避免混淆

## 二、实施方案

### 1. 选择保留的配置源

**保留**：项目根目录的 `/.env` 文件

**原因**：
- `start.py` 在项目根目录运行，自然读取项目根目录的 `.env`
- 统一管理所有环境变量
- 便于部署时统一修改

### 2. 移除冗余的 `.env` 文件

**删除**：`/development_platform/.env` 文件

**原因**：
- 避免配置冲突
- `start.py` 已经通过 `env` 参数传递环境变量给Flask应用
- 保持配置源单一

### 3. 修改 `development_platform/run.py`

**变更**：移除 `load_dotenv()` 调用

**原因**：
- 环境变量由 `start.py` 通过 `env` 参数传递
- 避免重复加载和可能的配置冲突

### 4. 验证配置传递链路

**验证点**：
1. `start.py` 读取 `/.env` → `SERVER_IP=127.0.0.1`
2. `start.py` 通过 `env` 参数传递给 `run.py`
3. `run.py` 中的 `load_dotenv()` 被移除，使用传递的环境变量
4. `__init__.py` 中的 `os.getenv('SERVER_IP')` 获取到 `127.0.0.1`
5. 模板中的 `{{ SERVER_IP }}` 渲染为 `127.0.0.1`

```
├── development_platform/
│   ├── .env                   # ❌ 删除（冗余）
│   ├── run.py                 # ✅ 修改（移除load_dotenv）

└── start.py                       # ✅ 保持不变（读取/.env并传递）
```

### 环境变量传递链路
```
/.env (SERVER_IP=127.0.0.1)
    ↓
start.py (读取并设置到 env)
    ↓
run.py (通过 env 参数接收)
    ↓
__init__.py (os.getenv('SERVER_IP'))
    ↓
模板 ({{ SERVER_IP }})
    ↓
HTML (http://127.0.0.1:9090/...)
```

### 配置管理方式

**本地开发环境**：
```env
SERVER_IP=127.0.0.1
```

**部署到服务器**：
```env
SERVER_IP=100.100.135.127
```

只需修改项目根目录的 `/.env` 文件即可，所有模块自动使用新配置。
