## 二、实施方案

### 1. 创建环境变量配置文件

**新建文件：** `.env.example`（模板）和 `.env`（实际配置）

```env
# 服务器IP配置
SERVER_IP=100.100.135.127
STATIC_SERVER_PORT=9090
FLASK_SERVER_PORT=8465
```

### 2. 更新依赖文件

**修改文件：** `requirements.txt`

* 添加 `python-dotenv>=1.0.0`

### 3. 修改核心启动脚本

**修改文件：** `start.py`

**变更内容：**

* 导入 `load_dotenv`

* 移除 `get_host_ip()` 方法（Lines 91-109）

* 移除自动IP检测逻辑

* 使用环境变量 `os.getenv('SERVER_IP', 'localhost')` 替代硬编码IP

* 更新所有打印的访问地址使用环境变量

### 4. 修改HTML模板文件

**修改文件：** `overview/overview_code.html`

* 将 `http://100.100.135.127:8465` 改为使用JavaScript动态获取：`http://${window.location.hostname}:8465`

**修改文件：** `app/templates/base.html`

* 将 `http://100.100.135.127:9090` 改为：`http://${SERVER_IP}:9090`（通过Jinja2传递）

**修改文件：** `app/templates/risk_cot/base.html`

* 同上，使用Jinja2变量

### 5. 修改Flask应用配置

**修改文件：** `development_platform/app/__init__.py`

* 在创建app时加载环境变量

* 将SERVER\_IP传递给模板上下文

**修改文件：** `development_platform/run.py`

* 确保环境变量在启动时加载

### 6. 更新文档

**修改文件：** `README.md`

* 添加环境变量配置章节

* 说明 `.env` 文件的创建和使用方法

* 更新快速开始指南，包含环境变量配置步骤

**修改文件：** `development_platform/README.md`

* 同样添加环境变量配置说明

### 7. 添加.gitignore规则

**修改文件：** `.gitignore`

* 添加 `.env` 到忽略列表（防止敏感信息泄露）

* 保留 `.env.example` 作为模板

## 三、环境变量命名规范

| 变量名                  | 说明        | 默认值         |
| -------------------- | --------- | ------------ |
| `SERVER_IP`          | 服务器IP地址   | `localhost`   |
| `STATIC_SERVER_PORT` | 静态服务器端口   | `9090`        |
| `FLASK_SERVER_PORT`  | Flask应用端口 | `8465`        |

## 四、测试验证计划

1. **本地测试**：使用 `SERVER_IP=localhost` 验证功能正常
2. **部署测试**：使用 `SERVER_IP=100.100.135.127` 验证部署环境
3. **跨环境测试**：验证不同IP配置下的链接跳转和API调用
4. **文档验证**：确认新配置方法在文档中清晰说明

## 五、注意事项

* 所有端口号（9090、8465、8081）保持不变

* **LLM API地址（`cot_synthesis_service.py`）保持不变**

* 仅修改服务部署相关的IP地址配置

* 保持原有代码逻辑和文件结构

* 确保向后兼容性（提供默认值）
